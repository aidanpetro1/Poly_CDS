# Simple D1 workup

The D1 workup uses **o1a** as a screening test and **o1b** as
confirmation. Screen-positive patients proceed to confirmation;
screen-negative patients exit the workup with D1 ruled out
immediately. Both confirmation outcomes are terminal — a
positive confirmation phenotypes as `:a_D1` (D1 present), a
negative confirmation phenotypes as `:a_D1_absent_via_o1b`
(absent-with-confirmation, which is clinically distinct from
absent-by-screen for re-screen scheduling purposes).

## Observations

```yaml-vocab
observations:
  - id: o1a
    display: D1 screening test
    loinc: "00000-1"
    results: [neg, pos]
  - id: o1b
    display: D1 confirmation test
    loinc: "00000-2"
    results: [neg, pos]
```

## Phenotypes

```yaml-phenotypes
phenotypes:
  - id: a_D1_initial
    display: Pre-workup
    description: No D1-relevant data yet; awaiting screen.
  - id: a_D1_pending
    display: Screen-positive, pending confirmation
    description: D1 screen returned positive; confirmation order recommended.
  - id: a_D1
    display: D1 confirmed
    description: Both screen and confirmation positive — D1 present.
  - id: a_D1_absent_via_o1a
    display: D1 ruled out at screen
    description: Screen returned negative; workup ended without confirmation.
  - id: a_D1_absent_via_o1b
    display: D1 ruled out at confirmation
    description: Screen positive but confirmation negative; D1 absent.
```

## Workflow

```yaml-protocol
initial:
  phenotype: a_D1_initial
  order: order_o1a
  on:
    neg:
      terminal: a_D1_absent_via_o1a
      conclusion: disease_D1_absent
    pos:
      phenotype: a_D1_pending
      order: order_o1b
      on:
        neg:
          terminal: a_D1_absent_via_o1b
          conclusion: disease_D1_absent
        pos:
          terminal: a_D1
          conclusion: disease_D1_present
```
