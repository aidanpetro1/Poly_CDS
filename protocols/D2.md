# Simple D2 workup

Mirror of the D1 toy protocol with the D2 vocabulary.

## Observations

```yaml-vocab
observations:
  - id: o2a
    display: D2 screening test
    loinc: "00000-3"
    results: [neg, pos]
  - id: o2b
    display: D2 confirmation test
    loinc: "00000-4"
    results: [neg, pos]
```

## Orders

```yaml-orders
orders:
  - id: order_o2a
    display: Order D2 screen
    fhir:
      resource: ServiceRequest
      code: "00000-3"
  - id: order_o2b
    display: Order D2 confirmation
    fhir:
      resource: ServiceRequest
      code: "00000-4"
```

## Conclusions

```yaml-conclusions
conclusions:
  - id: disease_D2_present
    display: D2 confirmed present
    icd10: E00.1
  - id: disease_D2_absent
    display: D2 ruled out
```

## Phenotypes

```yaml-phenotypes
phenotypes:
  - id: a_D2_initial
    display: Pre-workup
    description: No D2-relevant data yet.
  - id: a_D2_pending
    display: Screen-positive, pending confirmation
    description: D2 screen positive; confirmation recommended.
  - id: a_D2
    display: D2 confirmed
    description: Both screen and confirmation positive.
  - id: a_D2_absent_via_o2a
    display: D2 ruled out at screen
    description: Screen negative; workup ended.
  - id: a_D2_absent_via_o2b
    display: D2 ruled out at confirmation
    description: Screen positive, confirmation negative.
```

## Workflow

```yaml-protocol
initial:
  phenotype: a_D2_initial
  order: order_o2a
  on:
    neg:
      terminal: a_D2_absent_via_o2a
      conclusion: disease_D2_absent
    pos:
      phenotype: a_D2_pending
      order: order_o2b
      on:
        neg:
          terminal: a_D2_absent_via_o2b
          conclusion: disease_D2_absent
        pos:
          terminal: a_D2
          conclusion: disease_D2_present
```
